移动应用App测试方法汇总

移动应用一般分为三种类型
Web App（web应用）：实际就是一个web页面
Native App（原生应用）：移动端的原生应用
Hybrid App（混合应用）：介于原生应用和Web应用的一种形式

移动应用专项测试方法总结：

功能业务测试

和web端的测试一样都需要考虑功能业务的测试

交叉事件测试

交叉事件测试意思就是在操作应用的时候来电话或者来短信的情况
需要考虑到的场景具体有

  1. 多个APP同时在后台运行，并且交替使用，比如测试的是视频播放软件，将两个视频播放软件同时打开，并且交替使用，看视频是否能正常的播放
  2. 使用app时，切换WiFi模式
  3. 使用APP时来电话
  4. 使用APP时来短信
  5. 使用app时，进入低电量模式
  6. 使用app时，系统提示更新的情况

兼容性测试

兼容性测试在app测试中是比较重要的一块，如果是web测试的话，只需要考虑到浏览器的兼容性就可以了，但是app测试，我们不能保证每个用户使用的都是同样的手机，所以在考虑手机的兼容性测试的时候一定要全面，主要包含以下几点：

  1. 操作系统的兼容性，ios和Android
  2. 设备的分辨率的兼容性
  3. 不同网络设备下的兼容性

兼容性测试可以借助第三方云平台来测试，国外的SauceLab，国内的Testin

流量测试

为啥要进行流量测试，因为使用app时会消耗流量，流量消耗太大会导致加载缓慢，流量测试需要考虑到以下几点

  1. 执行业务操作使用的流量
  2. 下载安装包所使用的流量情况
  3. app购买或升级需要的流量
  4. app在后台运行时的流量情况

对于流量的使用情况可以利用 tcpdump、Wireshark 和 Fiddler 等网络分析工具。

耗电量测试

耗电量测试通常从三个方面来考量：

  1. app运行但是没有操作业务时的耗电情况
  2. app运行并且频繁操作业务的耗电情况
  3. app后台运行的耗电情况

对于Android 可以通过命令行：Android 通过 adb 命令“adb shell dumpsys battery”获取app的耗电情况。
对于ios通过Apple的官方工具 Sysdiagnose 来收集耗电量信息。

弱网测试

平时在公交，地铁使用手机应用的时候，经常会数据丢包的情况，所以我们测试的时候尽可能的模拟出这样的网络环境，及时发现问题，在上线之前就能规避这种因为弱网环境出现的问题。
推荐开源的移动网络测试工具：FacebookAugmented TrafficControl（ATC）

边界测试

边界测试是指在使用app时，临界的情况，比如：

  1. 系统内存占用大于 90% 的场景
  2. 飞行模式来回切换
  3. 长时间使用app的情况
  4. 系统的时间早于或者晚于标准时间

安装更新卸载测试

首先我们来看安装测试

  1. 安装时内存不足
  2. 安装时出现意外重启、断电、断网的情况
  3. 安装后是否能正常运行
更新测试

更新app分为强制更新和非强制更新

  1. 非强制更新时，是否可以取消更新，取消更新后再次打开app时是否还是会提示更新
  2. 非强制更新，点击确定更新后，是否更新成功
  3. 强制更新，用户退出后，再次运行看是否还有提示更新
  4. 更新成功后，数据配置、状态是否正常
  5. 直接卸载后安装app,看是否有更新
  6. 升级时出现意外情况断电、重启、断网

认识selenium

selenium一系列基于web自动化测试工具，selenium包含了四种组件，描述如下
|组件名称|描述|
|--------:----:|
|selenium IDE|是Firefox的一种插件，用于录制脚本动作，可转换为脚本语言|
|selenium RC|selenium远程控制，并支持Java，C＃，PHP，Python和Ruby |
|selenium webdriver| Selenium2.0有简洁的AP，WebDriver和WebElement对象，更好的抽象。且支持多中操作系统，多语言，多浏览器|
|selenium grid|可以对测试脚本级做分布式处理，即可以在不同的平台下执行测试用例|

selenium基础

selenium 命令——selense

  1. actions:操作的具体动作，点击链接等。
  2. accessors：检查系统的当前状态
  3. assertions：断言，判断预期结果和实际结果是否相符，断言又分为三种，assert、verify、waitFor
    assert：当assert失败时，测试将终止。
    verify: 当verify失败时，测试还会继续。
    waitFor: 等待某个条件为ture时执行，用于测试AJAX程序。

selenium环境设置

  • 1、java环境设置，因为selenium工具是由java语言开发的，因此需要配置java环境
  • 2、安装MyEclipse开发工具
  • 3、安装Firefox中的插件，firebug、firepath、selenium IDE.
  • 4、下载selenium RC 服务jar包，我的是selenium-server-standalone-2.40.0.jar
  • 5、下载selenium webdriver 依赖包 selenium-java-2.45.0.zip

启动selenium RC服务

使用cmd命令跳转至存放selenium-server-standalone-2.40.0.jar 的位置，执行java - jar selenium-server-standalone-2.40.0.jar 正常启动服务。
可以使用简单的方法来完成启动服务这个操作
在桌面新建一个txt文本，内容cd D :\selenium java - jar selenium-server-standalone-2.40.0.jar，将文件后缀改为bat，每次启动时，双击即可。

selenium RC的工作原理

selenium RC的组成

1.selenium server 代理服务器

  • selenium server 主要用于控制浏览器行为

2.Client Libraries 客户端

  • 通过测试案例用于控制selenium server行为
  • 提供的编程语言包括java、c#、Perl、Python、PHP

3.selenium RC 工作步骤

  • 通过客户端库给selenium server发送请求，请求与selenium server建立连接
  • selenium Server的Launcher 启动浏览器，将selenium core 加入到浏览器中，并且将浏览器的代理设置为selenium server 的代理 Http Proxy
  • 客户端库通过向selenium server发送http请求后，selenium server接收到请求后进行解析，然后通过Http Proxy 发送js命令通知selenium core 执行浏览器动作

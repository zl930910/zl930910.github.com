定位页面元素

页面定位元素包含基本的8中方法，包含

  • By.className(className))
  • By.cssSelector(selector)
  • By.id(id)
  • By.linkText(linkText)
  • By.name(name)
  • By.partialLinkText(linkText)
  • By.tagName(name)
  • By.xpath(xpathExpression)

下面是对各种定位方法的使用总结
（1）、根据id定位元素，在html标签中id是唯一的，不能重复的

（2）、根据name定位元素，name允许多个存在，重复

（3）、根据class属性定位元素、class允许重复

（4）、根据标签名称定位元素,tagName在网页的元素中可能存在多个所以返回的是一个集合

（5）、根据css元素定位元素，注：css包含多种选择器，可根据情况选择进行定位，具体请查看http://www.w3school.com.cn/cssref/css_selectors.asp，选择器也允许重复的出现哦

（6）、通过超文本连接来定位元素

（7）、通过超文本模糊匹配定位元素

（8）、通过xpath定位元素，xpath是 XML path的简称，可通过http://www.w3school.com.cn/xpath/xpath_syntax.asp学习语法

注：由于在html中定位元素时，有些属性内容是允许重复的，要想准确的定位到自己需要的元素，建议还是使用By.id或者By.Xpath获取元素，另外，使用Xpath时需要慎重，当开发者频繁的改动网页结构时，Xpath就必须要重新定位。

  1. 为什么Firefox就可以准确的定位到元素，而使用IE9和chrome浏览器就不能准确的定位？
    报错信息：
    Exception in thread “main” org.openqa.selenium.NoSuchElementException: Unable to find element with id == alertDialogs (WARNING: The server did not provide any stacktrace information)
    Command duration or timeout: 324 milliseconds
    问题原因：由于在还没有等浏览器加载完成就开始定位元素，肯定定位不了的。
    解决方案：在定位元素前需要设置等待时间，等待页面加载完成后，开始定位.
